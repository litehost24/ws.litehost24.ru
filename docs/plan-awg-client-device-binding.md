# План: Android-клиент (fork wgtunnel) + 1-клик подключение по App Links (без привязки устройства)

## Цели
- Один Android-клиент (наш форк) на базе `wgtunnel` (MIT), работает только с AmneziaWG.
- Открытие приложения с сайта по HTTPS App Links с параметрами `subscription_id` и `t` (токен).
- Приложение само скачивает `peer-1.conf`, импортирует/обновляет профиль и включает VPN.
- Без отдельного шага "привязать устройство".
- Токен одноразовый (single-use) и живёт 10 минут (чтобы успеть установить APK и нажать кнопку ещё раз).

## UX
1. Пользователь в `/my/main` на нужной VPN-карточке жмёт `Открыть в приложении`.
2. Открывается приложение WS VPN по App Links `https://ws.litehost24.ru/app/open?...`.
3. Приложение по токену скачивает `peer-1.conf` для конкретного `subscription_id`.
4. Приложение импортирует/обновляет туннель и включает VPN (первый раз Android спросит системное разрешение VPN).
5. Если приложения не было установлено:
   - пользователь скачивает APK со страницы fallback
   - возвращается в кабинет и нажимает кнопку ещё раз (токен TTL 10 минут).

## Про шаринг (важно)
- Специальной защиты от шаринга конфига/доступа не делаем.
- Если один и тот же `peer-1.conf` используют одновременно на двух устройствах, возможны постоянные реконнекты и плохая связь (endpoint "перетягивается").

## Модель данных (Laravel)
### Таблица `app_connect_tokens`
- `id`
- `token_hash` (sha256)
- `user_id`
- `subscription_id` (конкретный VPN-уровень/карточка)
- `expires_at` (TTL 10 минут)
- `used_at` (nullable)
- timestamps

Токен:
- одноразовый (после выдачи конфига ставим `used_at`)
- живёт 10 минут

## Серверные эндпойнты
### Открытие приложения
- `GET /app/open?subscription_id=...` (web, auth)
  - создаёт `CONNECT_TOKEN`
  - открывает/редиректит на `https://ws.litehost24.ru/app/open?t=...&subscription_id=...`
- `GET /app/open?t=...&subscription_id=...` (web, public)
  - fallback страница: "Открыть приложение" + "Скачать APK" + короткая инструкция

### Выдача конфига для приложения
- `GET /api/app/config?t=...&subscription_id=...` (public, token-gated)
  - проверяет TTL/used/subscription_id
  - выдаёт `peer-1.conf`
  - помечает токен использованным

Источник конфига:
- из zip (`peer-1.conf` внутри `file_path` последней активной записи `UserSubscription` для `user_id + subscription_id`).

## Логика "конфиг поменялся"
- При каждом открытии по ссылке с сайта приложение всегда получает актуальный `peer-1.conf`.
- Если пользователь открыл приложение без сайта, синхронизация конфига не выполняется (сознательно, чтобы не вводить постоянные токены/привязку).

## Ошибки
- токен истёк/уже использован: показать "Ссылка устарела. Откройте кабинет и нажмите кнопку ещё раз".
- подписка не активна или конфиг не найден: показать понятную ошибку и ссылку в кабинет.

## Android (fork)
- База: `wgtunnel/wgtunnel` (MIT), ребрендинг под WS VPN.
- App Links для `https://ws.litehost24.ru/app/open`.
- При открытии по ссылке: скачать конфиг по `t + subscription_id`, импортировать/обновить, включить VPN.

## Сборка APK на сервере (без GitHub/GitLab)
Рекомендуется настроить git (даже если репозиторий только локально на сервере), чтобы:
- обновлять код одной командой (`git pull`)
- облегчить переезд на другой сервер (`git clone` + secrets + Android SDK/JDK)

Сборку запускать вручную (не cron), чтобы не нагружать сайт в пиковое время.

Путь публикации APK на сайте:
- `/home/admin/web/ws.litehost24.ru/public_html/storage/files`
